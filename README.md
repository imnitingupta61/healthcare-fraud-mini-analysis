# Healthcare Fraud Mini Analysis

## Objective
Analyze healthcare claims data to identify anomalies and suspicious patterns using SQL.

## Key Features
- Provider-wise average comparison
- Member-level risk segmentation
- Claim anomaly detection (Above/Below Avg)
- Multiple CTE and window functions

## Dataset
Includes:
- Claims
- Members
- Providers

## SQL Concepts Used
- JOIN
- CTE
- Window Functions
- CASE WHEN
- LAG
- Aggregations

## Key Insights
- Identified high-risk members
- Detected abnormal claim patterns
- Compared claims with provider benchmarks
